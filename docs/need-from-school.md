# 需要你提供的信息

下面这些不用一次性全给。我们先拿最小一批，就可以继续写真实接口。

## 1. OAuth / 统一身份认证字段

请确认登录后能拿到哪些字段：

- user_id 或学号
- 姓名
- 本科生 / 研究生 / 教师
- 学院
- 专业
- 年级
- 班级
- 校区
- 在校状态：新生、在校、毕业、休学等

也需要确认：

- OAuth 回调地址如何配置
- 是 CAS、OAuth2、企业微信、统一身份认证，还是学校自研接口
- 是否有测试账号或测试环境

当前已知正式域名：

```text
https://xgaigc.ahau.edu.cn
```

当前已拿到学生信息样例，详细记录见 `docs/oauth-ahau.md`。

## 2. 第一批办事事项

建议先提供 20 到 50 条高频事项，每条尽量包含：

- 事项名称
- 分类
- 适用对象
- 适用年级 / 校区 / 学院
- 办理入口
- 负责部门
- 所需材料
- 办理流程
- 注意事项
- 常见问法 / 关键词
- 来源页面
- 最后核验时间

## 3. Dify 信息

后面接 Dify 时需要：

- Dify 部署地址
- API Key
- 你希望先用 Workflow 还是 Chatflow
- 是否已经有学校制度、通知、办事说明文档

## 4. 手机端入口

需要确认第一版运行形态：

- H5
- 微信小程序
- 企业微信 / 学校 App WebView
- 先本地演示

## 5. 画像记忆边界

需要一起定哪些可以自动保存，哪些必须确认：

- 可自动保存：点击、搜索、收藏、最近意图
- 建议确认后保存：考研、就业、竞赛、资助、宿舍区等偏好
- 不建议传给 Dify：手机号、身份证号、完整学号、敏感身份信息
